# Spec Summary (Lite)

Productionalise `archive/dashboard-full-claude-design/` into a running React + TypeScript app: extract its single semantic token layer into Tailwind v4 theme config (`@theme inline`) with dark mode and a working toggle, extract the shared shell that is currently copy-pasted into all 17 screens, port the component vocabulary, and stand up routing, TanStack Query and OpenAPI-driven type generation. The design output is a one-time seed; after this, React is the source of truth. TSX not JSX, because the derived-metrics figures are legitimately nullable and a silently-undefined MPG rendering as a blank cell is the exact defect class this project exists to eliminate.

This spec is the **foundation**; the 17 screens themselves are specced in `docs/specs/2026-07-15-frontend-screens/`.

**Corrected 2026-07-15.** Written against the superseded single-screen concept `dashboard-design-idea/dashboard.html`, and wrong on two counts. It claimed a **two-layer** token system (raw palette → semantic) and asked for both to be ported: verified, **neither concept contains a raw-palette variable** — the palette lives only in the field manual and was inherited as hex values, not variables. There is one layer. And it said fonts stay inlined base64 in one breath and extracted to `.woff2` in another; **DEC-010** settles that as `.woff2` served from `'self'`, which was always what the CSP requirement actually needed.
